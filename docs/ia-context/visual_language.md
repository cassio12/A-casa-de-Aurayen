# Linguagem Visual — A Casa de Aurayen

> "Cada elemento visual deve parecer pertencer ao mundo de Aurayen."

---

# Objetivo

Este documento define a identidade visual da Casa de Aurayen.

Seu propósito é garantir consistência entre todas as telas, componentes, ilustrações e futuras implementações realizadas por pessoas ou por IA.

Toda decisão visual deve reforçar a sensação de que o visitante está explorando um lugar real.

---

# Identidade

Aurayen é uma academia ancestral.

Não é um software.

Não é um produto digital.

Não é um dashboard.

A interface representa objetos físicos existentes dentro desse mundo.

---

# Materiais

Toda a interface deve transmitir peso, textura e história.

## Pedra

Utilizada em:

- paredes
- pilares
- portais
- escadarias

Sensação:

- solidez
- permanência
- antiguidade

---

## Madeira

Utilizada em:

- portas
- estantes
- mesas
- vigas

Sensação:

- acolhimento
- tradição
- conhecimento

---

## Bronze e Ferro

Utilizados em:

- dobradiças
- brasões
- fechaduras
- molduras
- mecanismos

Sensação:

- resistência
- autoridade

---

## Pergaminho

Utilizado para:

- contos
- documentos
- cartas
- mapas

Nunca utilizar folhas brancas comuns.

---

## Vidro

Sempre antigo.

Preferencialmente:

- vitrais
- lentes
- telescópios

Nunca vidro moderno.

---

# Iluminação

Aurayen nunca parece iluminada por LEDs.

As fontes de luz são naturais.

Exemplos:

- tochas
- velas
- lareiras
- luar
- luz entrando pelas janelas

A iluminação cria atmosfera.

Não apenas visibilidade.

---

# Texturas

Superfícies devem apresentar pequenas imperfeições.

Exemplos:

- pedra desgastada
- madeira marcada
- pergaminho envelhecido
- metal oxidado

Evitar superfícies perfeitamente lisas.

---

# Tipografia

## Títulos

Podem transmitir elegância clássica.

Devem lembrar inscrições antigas.

Nunca exagerar na ornamentação.

---

## Corpo do texto

Leitura confortável.

Boa hierarquia.

Excelente contraste.

Priorizar legibilidade.

---

# Ícones

Os ícones devem parecer gravuras.

Evitar:

- estilo flat
- emojis
- material icons

Preferir:

- brasões
- gravuras
- símbolos
- runas
- selos

---

# Botões

Botões não parecem botões.

Eles representam objetos.

Exemplos:

✔ brasões

✔ selos

✔ placas

✔ inscrições

✔ mecanismos

Evitar:

❌ retângulos arredondados

❌ gradientes modernos

❌ efeitos chamativos

---

# Cartões

Aurayen evita cartões tradicionais.

Sempre que possível utilizar:

- livros
- pergaminhos
- quadros
- mesas
- nichos
- estantes

---

# Navegação

O usuário nunca deve sentir que mudou de página.

Ele deve sentir que atravessou um ambiente.

Exemplos:

Biblioteca

↓

Corredor

↓

Observatório

Nunca utilizar transições bruscas.

---

# Animações

Todas possuem propósito.

Movimentos:

- lentos
- suaves
- naturais

Exemplos:

porta pesada abrindo

poeira atravessando luz

tocha tremulando

livro abrindo

pergaminho desenrolando

---

# Áudio

O áudio constrói o ambiente.

Nunca compete com ele.

Exemplos:

vento

madeira

fogo

passos

páginas

chuva

---

# Elementos Narrativos

Sempre que possível, substituir componentes tradicionais por elementos do mundo.

| Interface tradicional | Aurayen |
|-----------------------|----------|
| Home | Salão Principal |
| Página | Ambiente |
| Menu | Mapa da Casa |
| Botão | Brasão |
| Card | Livro |
| Modal | Pergaminho |
| Link | Porta |
| Lista | Estante |
| Loader | Ritual de abertura |

---

# O que evitar

Evitar qualquer elemento que lembre:

- aplicativos SaaS
- dashboards
- landing pages
- templates prontos
- interfaces corporativas

---

# Princípio Final

Cada tela deve conseguir responder, sem texto:

> "Onde estou?"

E cada objeto deve responder:

> "Por que ele existe dentro de Aurayen?"

Se um elemento não possui um propósito narrativo, ele provavelmente não pertence ao projeto.
